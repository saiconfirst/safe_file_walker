# Contributing

We welcome contributions! Please follow these guidelines.

## How to Contribute

1. **Fork** the repository.
2. **Create a branch** for your feature or bugfix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes** and ensure they follow the project's code style.
4. **Test your changes** (see [Testing](#testing)).
5. **Commit your changes** with a descriptive commit message.
6. **Push** to your fork and open a Pull Request.

## Code Style

- Use **Black** for code formatting.
- Use **flake8** for linting (max line length 127).
- Use type hints where possible.
- Write docstrings for public functions.

## Testing

Currently, the project lacks comprehensive tests. If you add new functionality, please include appropriate tests.
